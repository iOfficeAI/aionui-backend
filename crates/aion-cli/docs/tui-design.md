# Aion CLI TUI 交互设计

## 概述

TUI 基于终端全屏渲染，提供类 IDE 的交互体验。核心场景：单聊、团队协作、指令操作。

---

## 视图模式

| 模式 | 触发 | 说明 |
|------|------|------|
| Chat | `aion` / `aion chat` | 单 agent 聊天 |
| Team | `aion team chat <ID>` / `/team` | 团队多 agent 协作 |
| Home | 首次启动 / `/home` | 选择 agent、最近对话、创建团队 |

---

## 键位绑定

### 全局键位

| 键位 | 操作 |
|------|------|
| `Ctrl+C` | 中断当前操作 / 退出 |
| `Ctrl+D` | 退出 Aion |
| `Ctrl+L` | 清屏 |
| `Esc` | 关闭弹出面板 / 返回上层 |
| `Tab` | 补全（指令/文件路径） |

### Chat 模式键位

| 键位 | 操作 |
|------|------|
| `Enter` | 发送消息（单行） |
| `Shift+Enter` | 换行（多行输入） |
| `↑` / `↓` | 浏览历史输入 |
| `Ctrl+U` | 清空当前输入 |
| `Ctrl+S` | 中断 agent 当前回复（stop） |

### Team 模式键位

| 键位 | 操作 |
|------|------|
| `Shift+↑` / `Shift+↓` | 切换聚焦的成员（左侧面板） |
| `Shift+←` / `Shift+→` | 切换会话视图（全局 / 单成员） |
| `Enter` | 发送消息到当前聚焦目标 |
| `@` + `name` | 指定消息接收成员 |

### Home 模式键位

| 键位 | 操作 |
|------|------|
| `↑` / `↓` | 选择列表项 |
| `Enter` | 确认选择 |
| `N` | 新对话 |
| `T` | 新团队 |
| `D` | 删除选中对话 |

## Team 模式会话视图

### 全局视图（默认）

显示所有成员的消息交错排列，按时间顺序。每条消息前标注发送者。

```
Leader: 任务分配完毕
→ Member-1: 收到
→ Member-2: 开始处理
You: 进度如何？
Leader: 正在汇总...
```

### 单成员视图（`Shift+←/→` 切换）

聚焦某个成员的独立对话流，只看该成员的输入输出。

```
[查看: Member-1]

You → Member-1: 处理下数据模块
Member-1: 好的，需要明确几个点...
Member-1: 已完成，输出在 output.json
```

## 流式输出

- agent 回复实时逐字渲染
- 光标位置显示 `▌` 表示正在输出
- 输出结束后光标消失
- Team 模式下多 agent 可并行输出，按接收顺序追加

## 指令面板

输入 `/` 触发指令面板：

```
┌───────────────────────────────┐
│ / 搜索指令...                  │
├───────────────────────────────┤
│ ❯ /agent     切换 agent 后端   │
│   /model     切换模型          │
│   /new       新对话            │
│   /history   历史对话          │
│   /team      团队操作          │
│   /mcp       MCP 列表         │
│   ─── Agent 指令 ───          │
│   /compact   压缩上下文        │
│   /think     扩展思考          │
├───────────────────────────────┤
│ ↑↓ 选择  Enter 确认  Esc 取消  │
└───────────────────────────────┘
```

- 模糊匹配过滤
- 分组显示（平台指令 / Agent 指令）
- 选中时右侧显示参数提示

## 消息气泡区域

### 消息类型渲染

| 类型 | 渲染方式 |
|------|---------|
| 纯文本 | Markdown 渲染（加粗、斜体、列表等） |
| 代码块 | 语法高亮 + 语言标签 |
| 图片引用 | `[Image #N]` 标签 |
| 文件引用 | `[File: path]` 可点击标签 |
| 工具调用 | 折叠区块，展示工具名 + 参数摘要 |
| 错误 | 红色标记 + 错误信息 |

### 发送者标识

| 来源 | 标识 |
|------|------|
| 用户 | `You:` |
| 单聊 agent | `{AgentName}:` |
| Team 成员 | `→ {MemberName}:` |
| 系统提示 | `[system]` 灰色 |

## 输入区域

### 单行模式（默认）

```
> 输入消息...█
```

### 多行模式（`Shift+Enter` 触发）

```
> 第一行内容
  第二行内容
  第三行内容█
                                              [Shift+Enter 换行 · Enter 发送]
```

### 文件/图片输入

- 粘贴文件路径：显示为普通文本，直接作为消息内容发送
- 粘贴图片（终端支持时）：显示 `[Image #N]`，作为附件上传

## 通知与提示

| 场景 | 表现 |
|------|------|
| agent 回复完成 | 终端 bell（可配置关闭） |
| team 成员状态变化 | 左侧面板图标更新（●/○/✗） |
| 网络断开 | 状态栏红色警告 `[disconnected]` |
| 指令执行成功 | 绿色一行提示后消失 |
| 指令执行失败 | 红色错误保留直到下次操作 |

---

## 成员状态图标

| 图标 | 状态 |
|------|------|
| `●` | Working（绿色） |
| `○` | Idle（灰色） |
| `◉` | Working + 有新输出（黄色） |
| `✗` | Error（红色） |

## 设计原则

1. **数据驱动**：TUI 只负责渲染后端推送的数据，不做业务逻辑
2. **响应式布局**：终端宽度不足时隐藏侧面板，自适应
3. **键盘优先**：所有操作可纯键盘完成，鼠标为辅助
4. **最小干扰**：流式输出时不打断用户输入，输入区独立
5. **渐进增强**：基础终端（无 true-color）退化为简单文本模式
